# CampOne 관리자 대시보드 - PRD (Product Requirements Document)

**버전**: 1.0
**작성일**: 2025-01-10
**목표**: 후보/캠프에게 전체 스코프를 5분 이내에 이해·시연하게 하는 관리자 대시보드

---

## 1. 제품 개요

### 1.1 비전
한 화면에서 브랜드·KPI·모듈 현황·알림을 보여주고, 각 모듈로 자연스러운 드릴다운이 가능한 선거 캠페인 관리 대시보드.

### 1.2 접근 전략 (5줄 요약)
1. **한 화면 이해**: 대시보드 홈에 브랜드 헤더·핵심 KPI·모듈 카드·알림을 집약
2. **데모 우선**: 각 모듈에 "랜딩 포스터"(요약 카피+핵심 가치+썸네일+CTA)
3. **공통 패턴**: KPI 헤더 → 필터 가능한 테이블/보드 → CRUD 폼 → 우측 패널
4. **브랜딩 일관**: 프라이머리 팔레트·타이포·아이콘·간격 스케일 고정, 라이트/다크 자동 대응
5. **즉시 구현**: Next.js+TS+Tailwind+shadcn/ui+React Query+MSW 스캐폴딩

## 3. 화면 정의

### 3.1 대시보드 홈 (Executive Overview)

**목적**: 후보 유해남 캠페인의 현황을 한눈에

| 영역 | 설명 |
|------|------|
| 상단 브랜딩 헤더 | 후보 로고·이름·슬로건 슬라이더(3종)·상태 배지(Active/Freeze/Archive) |
| KPI 카드 (6+1) | 활성 연락처, 메시지 발송/오픈율, 콘텐츠 퍼블리시, 이벤트/참석 목표, 모금 합계, 완료 태스크, 여론 트렌드 지수 |
| 모듈 바로가기 카드 | 5개 모듈 - 포스터 썸네일·핵심 지표 2~3개·CTA |
| 최근 활동 타임라인 | CRUD/설정/발송/승인 기록 (필터: 사용자/모듈/심각도) |
| 알림 센터 | 탭(시스템/워크플로우), 읽음처리/고정 |
| 하단 푸터 | 시스템 상태·버전·언어/테마 토글·퍼블릭 사이트 열어보기 |

### 3.2 모듈 공통 레이아웃

- 상단 Summary KPI (2~4개)
- 필터/정렬/검색 가능한 테이블 또는 칸반/캘린더 보드
- CRUD: 생성/수정/삭제(드로어), CSV 대량 업로드, 태그, 상태 토글
- 우측 고정 패널: 빠른 편집, 관련 오브젝트 링크드 인사이트
- 감사 추적: 레코드 단위 활동 로그

### 3.3 모듈별 주요 화면

| 모듈 | 경로 | 주요 기능 |
|------|------|----------|
| **M1 Insights** | /pulse | 이슈 레이더, 감성/타임라인, 지역 히트맵, 리스크 경보 |

## 5. KPI 정의

### 5.1 대시보드 홈 - 공통 KPI 6 + 트렌드 1

| KPI | 정의 | 계산식 | 임계값(색상) |
|-----|------|--------|-------------|
| 활성 연락처 수 | Opt-in된 연락처 수 | `contacts where optIn=true` | ▲증가: 초록 / ▼감소: 노랑(-1~5%)·빨강(-5%↓) |
| 메시지 발송수/오픈율 | 기간 내 발송량·열람 비율 | `발송수= Σ sent` `오픈율= opened/sent` | 오픈율 30%↑ 초록 · 15~30 노랑 · 15%↓ 빨강 |
| 콘텐츠 퍼블리시 | Studio 공개 건수 | `count(published)` | 주간 20↑ 초록 · 10~19 노랑 · 10↓ 빨강 |
| 이벤트/참석 목표 | 예정 이벤트 수·목표 합계 | `count(status in planning,scheduled) / Σ attendeeTarget` | 목표 달성률 80%↑ 초록 · 60~79 노랑 · 60↓ 빨강 |
| 모금 합계 | 기간 내 승인 후원액 | `Σ donations.amount` | 목표 대비 90%↑ 초록 · 70~89 노랑 · 70↓ 빨강 |
| 완료 태스크 | 완료 업무 수 | `count(status=done)` | 계획 대비 90%↑ 초록 · 70~89 노랑 · 70↓ 빨강 |
| 여론 트렌드 지수 | GT/NT/SNS 관심도 합성(0~100) | `Index = 0.30*GT + 0.30*NT + 0.40*SNS` | 전주 대비 +10%p↑ 초록 · −5~+10 노랑 · −5%p↓ 빨강 |

### 5.2 모듈 KPI 요약

- **Insights**: 합성 지수(0~100)·소스별 관심도·감성 지수·급증 경보 수·키워드 Top5
- **Studio**: 게시 준비율·퍼블리시 수·템플릿 재사용률·외부 채널 클릭
- **Policy Lab**: 공약 완성도 점수(0~100)·검토 대기·시민 질문 매칭률
- **Ops**: 체크리스트 완료율·SLA 준수율·미처리 알림·역할 승인 대기
- **Civic Hub**: 퍼널(sent→delivered→opened→replied)·세그먼트 커버리지·A/B 승자율·인바운드 처리시간

## 7. 상호작용 & 마이크로카피

### 7.1 대표 플로우 (채택)
**세그먼트 생성 → 메시지 템플릿 선택 → A/B → 예약 발송 → 성과 확인**
- 데이터 흐름이 3분 내 명확, 퍼널 시각화

### 7.2 확인/경고/에러 템플릿
- **확인**: "이 세그먼트(1,240명)에게 SMS를 발송할까요?"
- **경고**: "오픈율이 낮은 제목입니다. A/B 테스트를 권장합니다."
- **에러**: "예약 시간이 과거입니다. '보내는 시간'을 다시 설정하세요."

### 7.3 성공 토스트
- "세그먼트가 생성되었습니다."
- "캠페인이 예약되었습니다."
- "템플릿이 저장되었습니다."
- "이벤트가 게시되었습니다."
- "런북이 실행되었습니다."

### 7.4 온보딩 투어 (4스텝)
1. **브랜딩 헤더**: "유해남 캠페인 현황을 한눈에. 슬로건은 자동 순환합니다."
2. **모듈 카드**: "각 포스터의 '관리 열기'로 세부 관리로 이동하세요."
3. **알림 센터**: "시스템/워크플로우로 구분해 중요한 것만 확인하세요."
4. **빠른 생성**: "여기서 메시지·이벤트를 즉시 만듭니다."
